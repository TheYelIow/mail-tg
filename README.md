# tg-mail-bot

Принимает входящую почту через **Resend**, пересылает её в **Telegram** и
позволяет отвечать на письма прямо из Telegram (ответ уходит в том же треде).

- Хостинг: **Vercel** (Next.js App Router, serverless route handlers)
- Приём почты: **Resend Inbound** (`email.received` webhook)
- Хранилище тредов: **Upstash Redis**
- DNS: **Cloudflare** (MX-записи на домене / поддомене)

```
Письмо → Resend (MX) → webhook /api/inbound → Telegram
Telegram reply → webhook /api/telegram → Resend send (In-Reply-To) → ответ ушёл
```

## Как это работает

1. Письмо на `что-угодно@твой-домен` принимает Resend (по MX-записям).
2. Resend шлёт `email.received` на `POST /api/inbound`. Подпись проверяется через
   svix. Тело письма подтягивается через `resend.emails.receiving.get()`.
3. Бот постит письмо в твой Telegram-чат и запоминает связку
   `telegram_message_id → email thread` в Redis. Вложения пересылаются следом
   отдельными сообщениями (фото — как фото, остальное — файлом).
4. Жмёшь **✍️ Ответить** под письмом, наполняешь черновик текстом/файлами и
   нажимаешь **✅ Отправить** (или быстрый нативный **Reply** одним сообщением).
5. `POST /api/telegram` находит тред в Redis и отправляет ответ через
   `resend.emails.send()` с заголовками `In-Reply-To` / `References` — письмо
   попадает в исходный тред у собеседника.

## Как происходит ответ

Под каждым письмом есть кнопка **✍️ Ответить**. Основной сценарий — кнопочный
черновик, куда можно добавить сколько угодно текста и файлов, а потом отправить
одной кнопкой.

1. В чат прилетает письмо + его вложения (если были). Под письмом — кнопка
   **✍️ Ответить**.
2. Нажимаешь **✍️ Ответить** → бот включает «режим ответа» и присылает статус
   черновика с кнопками **✅ Отправить** и **❌ Отменить**.
3. Шлёшь в чат что угодно — текст, файлы, фото (по очереди или вместе). Всё
   копится в черновике; статус-сообщение обновляется и показывает текст и
   список имён вложений. Кнопка **🗑 Очистить файлы** убирает вложения, не
   трогая текст (появляется, только когда файлы есть).
4. Жмёшь **✅ Отправить** → бот скачивает файлы из Telegram, собирает письмо и
   отправляет. Либо **❌ Отменить** (или команда `/cancel`) — черновик сбрасывается.
5. Письмо уходит так:
   - **To** — адрес отправителя исходного письма;
   - **From** — адрес, на который письмо пришло (тот же ящик, что писал
     собеседник), с фолбэком на `MAIL_FROM`;
   - **Subject** — `Re: <исходная тема>`;
   - заголовки **In-Reply-To** / **References** — чтобы у собеседника ответ лёг
     в ту же цепочку.
6. Статус черновика меняется на `✅ Ответ отправлен на <адрес>`.

**Быстрый вариант без кнопок:** можно просто сделать нативный **Reply** на письмо
одним сообщением (с файлом или без) — оно уйдёт сразу.

Заметки про цвет кнопок и ограничения:

- Telegram Bot API **не поддерживает произвольный цвет inline-кнопок** — они
  всегда в стиле клиента. «Цвет» задаётся эмодзи в подписи (✍️ / ✅ / ❌).
- Чтобы кнопки работали, вебхук должен слушать `callback_query` — это уже
  включено в `npm run set-webhook` (`allowed_updates: ["message","callback_query"]`).
- Черновик — **один на чат** и живёт 1 час; если тред старше 60 дней (истёк в
  Redis) — ответить нельзя.

## Файлы и фото

Поддерживаются **в обе стороны**:

- **Входящие.** Вложения письма скачиваются по временной ссылке Resend
  (`download_url`, живёт ~1 час) и пересылаются в Telegram: картинки — как фото,
  остальное — как документ. Лимиты пересылки по URL: фото ≤ 5 МБ, файл ≤ 20 МБ;
  что крупнее — придёт пометкой со ссылкой на Resend.
- **Исходящие.** Приложи к своему Reply файл или фото — бот скачает его из
  Telegram (до 20 МБ) и прикрепит к письму через `attachments`. Resend разрешает
  до 40 МБ суммарно на письмо.

## Настройки бота в Telegram (@BotFather)

Минимум для работы — только токен. Остальное по желанию, для удобства:

- **Токен** — `/newbot` или `/token` → значение в `TELEGRAM_BOT_TOKEN`.
- **Команды меню** — `/setcommands`, затем вставь:
  ```
  start - как пользоваться ботом
  help - помощь
  ```
- **Privacy mode** — для личного чата неважно. Если захочешь использовать бота в
  группе и чтобы он видел все сообщения — `/setprivacy` → **Disable**.
- **Доступ.** Бот отвечает только чату из `TELEGRAM_CHAT_ID`; любые другие
  сообщения молча игнорируются. Регистрировать кого-то у BotFather не нужно.
- Вебхук ставится **не** через BotFather, а скриптом `npm run set-webhook`
  (см. ниже) — он же закрывает приём левых запросов через `secret_token`.

## Настройка

### 1. Resend — домен для приёма

1. В Resend → **Domains** добавь свой домен (или поддомен, напр. `mail.твой-домен`,
   чтобы не трогать существующую почту).
2. Resend покажет DNS-записи. В **Cloudflare** добавь:
   - **MX**-запись(и) приёма — значения берёшь ровно те, что показывает Resend.
   - **TXT (SPF)** и **DKIM (CNAME/TXT)** — нужны, чтобы исходящие ответы не падали
     в спам.
   - ⚠️ Для MX-записей в Cloudflare **Proxy выключен** (это DNS-only, оранжевое
     облако недоступно для MX — так и должно быть).
3. Дождись верификации домена в Resend (Verified).

### 2. Resend — inbound webhook

1. Resend → **Webhooks** → Add Endpoint.
2. URL: `https://ТВОЙ-АПП.vercel.app/api/inbound`
3. Событие: **`email.received`**.
4. Скопируй **Signing Secret** (`whsec_...`) → переменная `RESEND_WEBHOOK_SECRET`.
5. Создай **API Key** (`re_...`) с правами на отправку и чтение полученных писем
   → `RESEND_API_KEY`.

### 3. Telegram-бот

1. Создай бота у **@BotFather**, получи токен → `TELEGRAM_BOT_TOKEN`.
2. Узнай свой `chat_id` (спроси у **@userinfobot**, либо напиши боту и посмотри id
   в логах Vercel) → `TELEGRAM_CHAT_ID`.
3. Придумай длинную случайную строку → `TELEGRAM_WEBHOOK_SECRET`.
4. (Необязательно) В BotFather задай команды меню — `/setcommands`:
   ```
   start - как пользоваться ботом
   help - помощь
   ```

### 4. Upstash Redis

1. Создай базу на **upstash.com** (или через Vercel → Storage → Upstash / KV).
2. Возьми REST URL и токен → `UPSTASH_REDIS_REST_URL`, `UPSTASH_REDIS_REST_TOKEN`.

### 5. Деплой на Vercel

1. Залей репозиторий и импортируй проект в Vercel.
2. В **Project → Settings → Environment Variables** добавь всё из `.env.example`.
3. Задеплой. Проверь `GET /api/health` → `{ "ok": true }`.

### 6. Регистрация Telegram-вебхука

После деплоя зарегистрируй вебхук (один раз и после каждой смены секрета/URL):

```bash
PUBLIC_URL=https://ТВОЙ-АПП.vercel.app \
TELEGRAM_BOT_TOKEN=... \
TELEGRAM_WEBHOOK_SECRET=... \
npm run set-webhook
```

Скрипт ставит вебхук на `/api/telegram` с `secret_token` и показывает
`getWebhookInfo`.

## Проверка

1. Отправь письмо на `что-угодно@твой-домен` — оно должно прилететь в Telegram.
2. Сделай **Reply** на это сообщение, напиши текст — собеседник получит ответ в
   том же треде.

## Локальная разработка

```bash
npm install
cp .env.example .env   # заполни значения
npm run dev
```

Вебхуки приходят извне, поэтому для локального теста пробрось туннель
(`ngrok http 3000`) и временно укажи его URL в Resend и в `set-webhook`.

## Что нужно для старта (чеклист)

Собери эти значения и положи в Environment Variables на Vercel:

| Переменная | Откуда |
|---|---|
| `RESEND_API_KEY` | Resend → API Keys |
| `RESEND_WEBHOOK_SECRET` | Resend → Webhooks → твой endpoint (`whsec_…`) |
| `MAIL_FROM` | адрес-фолбэк отправителя, напр. `"Inbox <inbox@домен>"` |
| `TELEGRAM_BOT_TOKEN` | @BotFather |
| `TELEGRAM_CHAT_ID` | @userinfobot |
| `TELEGRAM_WEBHOOK_SECRET` | придумываешь сам (длинная случайная строка) |
| `PUBLIC_URL` | URL приложения на Vercel (для `set-webhook`) |
| `UPSTASH_REDIS_REST_URL` | Upstash / Vercel KV |
| `UPSTASH_REDIS_REST_TOKEN` | Upstash / Vercel KV |

Порядок действий: **домен в Resend + DNS в Cloudflare → API-ключ и webhook в
Resend → бот в Telegram → Upstash → деплой на Vercel с env → `npm run
set-webhook` → тестовое письмо.** Подробности — в разделе «Настройка» выше.

## Ограничения / заметки

- **Один пользователь.** Бот обслуживает только `TELEGRAM_CHAT_ID`; остальные
  чаты игнорируются.
- **Вложения** работают в обе стороны (см. «Файлы и фото»). Ограничения по
  размеру: входящие — фото ≤ 5 МБ / файл ≤ 20 МБ при пересылке по URL; исходящие
  — до 20 МБ из Telegram и до 40 МБ суммарно на письмо в Resend.
- HTML писем конвертируется в текст упрощённо (без внешних зависимостей).
- Треды в Redis живут 60 дней; Resend хранит полученные письма 30 дней.
